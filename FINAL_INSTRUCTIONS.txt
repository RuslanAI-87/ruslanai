ИНСТРУКЦИЯ ПО ИСПРАВЛЕНИЮ ПРОБЛЕМЫ С ВЕКТОРНЫМ ХРАНИЛИЩЕМ

Для решения проблемы с сообщением "Векторное хранилище недоступно. Используется базовая система памяти", 
следуйте этим инструкциям:

1. ЗАКРОЙТЕ все запущенные API-серверы или окна с Python (если они есть)

2. ЗАПУСТИТЕ скрипт fix_memory_system.bat
   Двойной клик по файлу fix_memory_system.bat 
   
   Этот скрипт:
   - Модифицирует файлы memory_system.py и vector_store.py для исправления проблем с кодировкой
   - Проверяет, что импорт библиотек работает корректно
   - Перезапускает API-сервер с правильными настройками

3. ПРОВЕРЬТЕ работу системы в веб-интерфейсе
   - Напишите сообщение с какой-то информацией (например: "Меня зовут Алексей")
   - Затем спросите что-то, требующее знания контекста (например: "Как меня зовут?")
   - Система должна правильно ответить, используя сохраненный контекст

4. ЕСЛИ ПРОБЛЕМЫ ОСТАЮТСЯ, проверьте:
   - Запущен ли API-сервер (должно быть окно командной строки с надписью "RuslanAI API with Vector Store")
   - Есть ли в логах сообщение "Векторное хранилище успешно импортировано" вместо предупреждения
   
5. ДОПОЛНИТЕЛЬНО:
   - Все необходимые библиотеки уже установлены (sentence-transformers, chromadb, torch)
   - В директории logs найдите файл vector_store.log для проверки деталей работы системы
   - Автоматически созданы резервные копии всех модифицированных файлов с расширением .bak

ВАЖНО: Для корректной работы системы ВСЕГДА используйте start_api_fixed.bat 
для запуска API-сервера в будущем.
